# Review: Task 015 — CLI — All 11 Commands

## Round 1

- [x] [process-revision-complete] **`stego fill create --slot <s>` does not detect ambiguous slot names.** The spec states: "Each component owns its own slot protos (no sharing between components -- duplication is cheaper than coupling)." This means multiple components can define slots with the same name. At `cmd/stego/main.go:271-282`, the implementation iterates `reg.Components()` (a Go map) and picks the first component whose slot list contains the given name. Map iteration order in Go is non-deterministic, so when two or more components define the same slot name, the command silently picks an arbitrary component — producing different `fill.yaml` `implements` values across invocations. The implementation should detect the ambiguity (count of matching components > 1) and return an error listing the matching components, so the user can disambiguate.

## Round 2

- [ ] **`stego fill create` ambiguous-slot error suggests `--component` flag that does not exist.** At `cmd/stego/main.go:292`, the error message for ambiguous slot names instructs the user: "specify --component to disambiguate." However, the `runFillCreate` function's FlagSet at line 225-226 only defines `--slot` — no `--component` flag exists. The user cannot follow this guidance. Passing `--component comp-a` would either be captured by the manual flag parser (lines 232-239) as a flag argument with `comp-a` as its value, causing `fs.Parse` to fail with an "unknown flag" error, or — depending on argument ordering — be treated as a positional argument (the fill name). Either way, disambiguation is impossible. The ambiguity detection itself works correctly (per round 1 fix), but the recovery path is broken: the implementation detects the problem and tells the user what to do, but the suggested action doesn't work. Either implement the `--component` flag in the FlagSet and wire it into the component selection logic, or change the error message to suggest a working alternative (e.g., qualifying the slot name as `--slot <component>.<slot>` and adjusting the matching logic, or removing the suggestion entirely and just listing the components).
