# Task 006: Generator Interface & gen.Context

**Spec Reference:** "Code Generation Mechanism", "Generated Code Structure"

**Status:** `complete`
**Review:** [specs/reviews/task-006.md](../reviews/task-006.md)

## Description

Define the `Generator` interface and supporting types that components implement.

- `Generator` interface: `Generate(ctx gen.Context) ([]gen.File, error)`
- `gen.Context` — carries resolved archetype conventions, entity definitions, expose blocks, slot bindings, and any component-specific config
- `gen.File` — path (relative to output), content (bytes), header (generated-by comment)
- `Wiring` struct — imports, constructors, routes that each component returns for shared file assembly
- Output namespace enforcement: each component declares `output_namespace`, generator can only write files under it
- Generated file header: `// Code generated by stego. DO NOT EDIT.`

## Spec Excerpt

> Components are Go packages implementing a `Generator` interface. Code is generated programmatically (`fmt.Fprintf` to a buffer + `go/format.Source()` for formatting), not via text templates.
> Each component declares an `output_namespace` and can only write files under it.

## Acceptance Criteria

- `Generator` interface defined
- `gen.Context`, `gen.File`, `Wiring` types defined
- Namespace validation rejects files outside declared namespace
- Generated file header enforcement
- Unit tests for namespace validation
